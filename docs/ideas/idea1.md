\section{Introduction}
Speculative decoding is a powerful inference optimization technique for large, autoregressive language models. It uses a smaller draft model to propose several candidate tokens, which are then verified by the larger target model in a single forward pass \citep{chen2023accelerating}. If the drafted trajectories match the target model's predictions, several tokens are accepted per iteration, drastically reducing inference latency. As such, advancements in speculative decoding make are important for efficient deployment of larger and more capable models.

\section{Problem}
In vanilla speculative decoding, the draft model produces a single token sequence for the target model to verify. Tree-based speculative decoding extends this by allowing the drafter to generate a tree of candidate sequences. The target model then verifies the entire tree in parallel, which allows multiple potential trajectories to be evaluated at once and further reduces the inference latency  \citep{10.1145/3620666.3651335}. Recent research on tree-based methods has shown that pruning depth and width of the drafter's tree can yield significant latency reductions \citep{zhang2026learning}. However, while prior work dynamically prunes the tree depth \textit{during} generation, it only prunes the width \textit{after} the tree is constructed. This lack of parallelism wastes valuable drafter compute on branches that are ultimately discarded. Thus, we aim to answer the following question: how can we train a single, unified policy to dynamically prune both the depth and width of individual branches \textit{during} generation to optimize decoding throughput?

\section{Status Quo}
Past work on optimizing tree-based speculative decoding underscores the latency benefits of pruning the drafter tree. One prominent line of work is OPT-Tree \cite{wang2025opttreespeculativedecodingadaptive}, which employs a dynamic programming algorithm to compute the optimal width and depth of the drafter tree at every decoding step. This approach improves the length of accepted sequences over using static hyperparameters for tree construction, yet the repeated invocation of the dynamic programming algorithm imposes substantial computational overhead, limiting its practicality for realtime inference.\newline\newline 
Expanding on this, Learning to Draft (LTD) \citep{zhang2026learning} replaces the expensive optimization loop with a two lightweight policies that are trained to maximize throughput. A depth policy dynamically decides whether to continue expanding the draft tree, while a size policy statically selects how many candidate tokens to verify after the full tree has already been generated. This dual-policy approach yields significant throughput gains, accelerating inference by 2.24x to 4.32x and improving upon prior state-of-the-art tree-based methods by up to 36.4\%.\newline\newline
Despite these advances, LTD presents three key limitations. First, while depth is pruned dynamically, the width policy operates only after tree construction is complete, resulting in unnecessary extra calls to the drafter model. Ablation experiments in the LTD paper confirmed that this second policy delivered only marginal throughput gains beyond those from the depth policy. Second, both policies are implemented as simple multi-layer perceptrons (MLPs), which restricts representational capacity when modeling the draft frontier. Third, training relies on Proximal Policy Optimization (PPO), which is highly sample-inefficient.
\section{Plan}
To address these gaps, \textbf{we propose to train a unified policy that dynamically prunes both depth and width \textit{during} generation}. We will explore both on-policy methods (like PPO) and offline methods (like Deep Q-Networks) to identify more sample-efficient training strategies, and evaluate various policy architectures to understand the trade-off between policy complexity and inference throughput.

We will use LTD as our baseline. Our first step is to modify its released code for use with Modal and reproduce its results on Qwen3.5-27B using Modal. Then, we shall experiment with our unified policy. We will train our policy using the HumanEval dataset and evaluate using  MT-bench, GSM8K, Alpaca, and Natural Questions as these datasets were used in the LTD paper \citep{zhang2026learning}. Our metrics will be the same as LTD's, namely the end-to-end inference speedup ratio relative to vanilla autoregressive decoding, and the average acceptance length per draft-and-verify cycle.
